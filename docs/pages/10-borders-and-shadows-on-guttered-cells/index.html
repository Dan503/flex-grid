Body:Borders and shadows on guttered cells

Mobile warning

This documentation is best read on a desktop/laptop computer for the sake of viewing the live demos at all screen sizes.

The demos on this page will not display on IE8 due to IE8's lack of support for the box-shadow property.

If a grid has gaps, trying to add borders or shadows to it directly won't work

The reason it doesn't look right is because Gutter Grid uses invisible borders to create the gaps. When you add shadows to the grid cells, the shadows also have to wrap around the gaps.

The grid__inner technique can be used to help with this problem. Whenever you want the child element of a grid cell to take up the same dimensions as the content area of that cell, you can use grid__inner / $inners:true.

However if you add a wrapper element with overflow hidden like we have been doing so far, the cell shadows will get cut off.

If you leaving off the wrapper though, it will causes unwanted space to appear at the bottom of the grid.

Instead (if using the class system) use a grid__paddedWrapper class on the wrapping element. This class adds padding-bottom: 1px to the wrapper. This will prevent the issue from occurring and allow for a visible overflow. The down side is that it will leave a 1px gap at the bottom of the grid. This gap is generally very difficult to notice though. If using the mixin system, you will need to add your own 1px of bottom padding to a wrapper element.

Using box shadow as a fake border

You might have noticed in the first example that I was able to add something that looked a lot like a border to each grid cell without needing to user inner elements. The secret to that is using a solid inset box-shadow.
